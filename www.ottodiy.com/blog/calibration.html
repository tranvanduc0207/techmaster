how to calibrate your own robot?

for precise walk and dance movements

· STEAM,Ottobuilder

In this post we will cover the whole topic that is driving a lot of uncertain among builders if they are actually assembling in the right way their robots but first what is calibration? As Wikipedia says:

"Calibration is the comparison of measurement values delivered by a device under test with those of a calibration standard of known accuracy. Such a standard could be another measurement device of known accuracy, a device generating the quantity to be measured such as a voltage, sound tone, or a physical artefact"

In our case will be "the calibration of the 4 servo motors"

If you understand the definition you will find out that you do NOT need to calibrate your Otto to be able to work properly because the calibration is ONLY for precise movements like let's say "straight walk" or standing in one leg without falling.

Why my Otto legs and feet are twisted?

It is a very frequent question in our Otto builder community and emails, Many Otto builders think they need to calibrate their robot because after assembly they faced this problem, but the true is that, they might skip the important warning in the instruction manual while putting the motors.

Otto uses 180 degrees servo motors that means the rotation is intentionally limited (we don't want the legs of Otto spinning like freewheels), if you see above picture, the first step is to make sure the servo motor is properly in the middle of the 180 degrees rotation (90 degrees) you don't have to and you won't be that precise at this point (a tolerance of 5 degrees is fine) but is absolutely necessary to put the white servo key in the center otherwise the foot in this case won't turn from side to side.

The same principle applies to the legs and to Otto PLUS metal gear motors.

Give a try! and now probably you discover that Otto is walking and dancing pretty good, and the rest of this post is not important to read at all!

You are right! In fact, i have only calibrated Otto once because with the proper assembly is not needed. But robots are precision machines and sometimes the servos will go crazy with random movements due use, that means we will have to electronically calibrate the servos to "zero" the position.

Since we have many ways to program Otto you will find out that is the same for calibration there are many ways some more difficult than others:

  • Arduino code manually (adjusting and iterating the TRIM values)
  • Arduino code automatic (saving in EEPROM memory the values by uploading this code)
  • mblock code (just use the example we already created
  • Bluetooth APP (connect to your Android smartphone, modify the parameters and see the changes)

In this post we will focus on the mblock way since is easier for most starter builders:

Otto calibration with mBlock

mBlock V3 is an opensource software released by MakeBlock (Makeblock) for easy programming Robot by Drop and Drag blocks. Based on Scratch 2.0 (MIT), beside sharing the great Feature of Scratch, mBlock also intergrate an Arduino Code generator for upload program into Arduino based Hardware.

 

OttoDIY mBlock Extension is made for Otto can be programmed in Scratch Language. This make Otto to be a great lowcost opensource robot for exploring STEAM.

Before start make sure you have the servos connected in the right place. 2, 3, 4, 5 ports

installing mBlock

 

Download mBlock here . Install the downloaded file and Open the software, now is time for adding the Otto extension:

Step 1: Go to Manage Extensions.

Step 2: Search OttoDIY Extension in Tab

View Available Extension on mBlock Server for download, Search OttoDIY and Click Download See Otto blocks appear in Scripts Area.

Step 3: Prepare for Coding

Otto Extension now only programming in Arduino mode, this mean the Scratch code will auto-Generate to Arduino Code for Uploading to Otto Board (like Arduino IDE). For Enter Arduino Mode, Go to Edit/Arduino Mode

Now you can easily program like assemply graphical blocks together, no longer to write script code like before. The Arduino code will auto-generated by Software and just upload to your Otto hardware.

Step 4: Prepare to Upload

Otto using Arduino Nano V3, Please select Hardware is Arduino Nano and Select right Serial Port for Upload (same Arduino IDE). Select Connection port for Upload program

Otto uses Arduino Nano as brain therefore we pick Arduino Nano (mega328) for programming

make sure you Otto appears in the serial port

Calibration example

Download example program from our Otto Scratch examples

Step 1: Select Clone/Download Zip, select folder to download and extract the Zip file to this folder

Step 2: Go to the Examples/Calibration folder and open the mBlock project file Calibration_mBlock_Program.sb2

Step 3: Select Edit/Arduino mode to prepare the Upload of this program to Otto

Step 4: Connect Serial Port to Otto and Upload program to Otto

After Upload program successfully,

Step 5: Connect Serial Port again to go into Serial Monitor mode, NOTE select Char Mode to display data in Text style

Step 6: Easy calibrate, Put the Otto facing to you, let it inverse side, Head down to table.

  • Typing a or z and Click Send to Adjust Left Leg
  • Typing s or x and Click Send to Adjust Left Foot
  • Typing k or m and Click Send to Adjust Right Leg
  • Typing j or n and Click Send to Adjust Right Foot
  • After all Legs and Foots in home position, all the adjustment angle are saved to EEPROM, Otto will automatically load these parameter for any future performance in the initialization.
